# Клиент-серверное приложение для обмена сообщениями

Проект представляет собой клиент-серверное приложение на C++ с использованием библиотеки Boost.Asio для сетевого взаимодействия. Сервер обрабатывает входящие запросы от клиентов, выполняет простую обработку (возвращает заранее определённые ответы) и отправляет результат обратно. Клиент может отправлять два типа сообщений: `signal` (содержит вектор комплексных чисел) и `information` (содержит количество ядер). Обмен данными осуществляется через кастомный транспортный протокол на основе TCP.

│ └── lib/ServerRequestResponseHandler/         # Обработчик запросов сервера
├── core/                                       # Общие библиотеки

│   ├── MessageHandler/                           # Парсинг/сериализация сообщений (JSON <-> объекты)

### Краткое описание модулей

- **ConnectionHandler** – предоставляет классы для установки TCP-соединений: `listen()`, `accept()`, `connect()`.
- **TransportHandler** – отвечает за отправку и чтение сообщений по TCP с использованием простого протокола: магическое число (4 байта), длина JSON-данных (4 байта), сами данные (JSON). Извлекает поле `type` из JSON и сохраняет его в структуре `TransportMessage`.
- **MessageHandler** – преобразует `TransportMessage` в объекты сообщений (`SignalMessage`, `InformationMessage`) и обратно. Использует фабрику (`CreatorMessage`) для создания объектов по типу сообщения.
- **RequestResponseHandler** – абстрактный интерфейс для обработки запросов и формирования ответов. Реализован в серверной части как `ServerRequestResponseHandler`.
- **ThreadPool** – пул потоков на основе `boost::asio::io_context`, позволяющий асинхронно выполнять задачи.
- **Server** – основной серверный класс: принимает подключения, для каждого клиента создаёт задачу в пуле потоков, обрабатывает запрос и отправляет ответ.
- **Client** – клиентский класс: подключается к серверу, отправляет запрос (в зависимости от введённого пользователем кода) и выводит полученный ответ.

## Типы сообщений

Обмен данными между клиентом и сервером осуществляется в формате JSON. Каждое сообщение обязательно содержит поле `"type"`, определяющее его тип. В зависимости от типа, сообщение может содержать дополнительные поля.

### Signal (сигнал)

Сообщение типа `"signal"` используется для передачи комплексных сигналов (вектора комплексных чисел).

#### Запрос (Request)
От клиента к серверу. Поле `"signal"` пустое, `"central_Freq"` может быть любым (сервер игнорирует его значение в текущей реализации).

```json
{
  "type": "signal",
  "central_Freq": 0,
  "signal": []
}
```
#### Ответ (Response)

От сервера к клиенту. Содержит заполненный массив `"signal"`, где каждый элемент — массив из двух чисел: действительной и мнимой части комплексного числа.

```json
{
  "type": "signal",
  "central_Freq": 6100,
  "signal": [[-88.65925598144531, -65.49491882324219]]
}
```

### Information (информация)

Сообщение типа `"information"` используется для передачи простых числовых параметров.
#### Запрос (Request)

Клиент отправляет запрос с `"numberCore"` равным 0 (условное значение, обозначающее запрос).
```json
{
  "type": "information",
  "numberCore": 0
}
```

#### Ответ (Response)

Сервер возвращает актуальное значение `"numberCore"`.

```json 
{
  "type": "information",
  "numberCore": 4
}
```
